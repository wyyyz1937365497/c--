# 2. 实验内容

## 2.1 二叉树的非递归遍历

### 2.1.1 问题描述

给定一组由 push 和 pop 组成的栈操作序列，要求根据操作过程恢复出对应的二叉树，并输出该二叉树的后序遍历序列。

### 2.1.2 基本要求

程序需要根据中序非递归遍历时栈的操作规律，唯一确定二叉树的结构，并输出最后得到的后序遍历结果。

### 2.1.3 数据结构设计

程序中的结点结构 Tree_node 保存当前结点字符、左孩子指针和右孩子指针。

为了模拟栈操作，自定义了 Tree_stack 类，内部用动态数组保存栈元素。每个栈元素 stack_node 记录当前结点指针以及附加状态。

在建树过程中，额外使用 last_pop 记录最近一次弹出的结点。这样在新的 push 到来时，可以区分当前结点应连接为父结点的左孩子，还是作为上一个弹出结点的右孩子。

### 2.1.4 功能说明（函数、类）

- Tree_node：表示二叉树结点，保存字符内容和左右孩子指针。
- stack_node：表示栈中的一个元素，包含结点指针和状态信息。
- Tree_stack：自定义栈结构，负责 push、pop 和获取栈顶结点。
- split(const string &str)：将一行输入按空格拆分成若干字段，便于识别 push 和 pop 指令。
- Tree_stack::push(Tree_node *tree_node, int staute)：将结点压入栈中。
- Tree_stack::pop()：弹出栈顶结点。
- Tree_stack::get_top()：返回当前栈顶元素。
- post_order(Tree_node *node, string &result)：递归后序遍历二叉树，并把结果追加到字符串中。
- delete_tree(Tree_node *node)：递归释放整棵树的动态内存。
- main()：读入操作序列，恢复二叉树，输出后序遍历结果。

### 2.1.5 调试分析（遇到的问题和解决方法）

该题的难点在于 push 和 pop 的对应关系并不是直接给出父子结点，而是要从栈状态中推断当前结点挂接到哪里。调试时重点处理了两种情况：

1. 栈非空时，新 push 的结点应优先作为当前栈顶结点的左孩子；
2. 如果前一步是 pop，则当前 push 的结点应作为最近弹出结点的右孩子。

通过维护 last_pop 指针，程序可以稳定地区分这两种情况，避免把左右孩子挂错位置。最终用后序递归输出结果，与样例数据一致。

### 2.1.6 总结和体会

本题让我更清楚地理解了“栈操作序列”和二叉树结构之间的关系。实现过程中，关键不在于后序遍历本身，而在于如何从操作序列恢复树形结构。该程序的时间复杂度为 O(n)，空间复杂度也为 O(n)，其中 n 为结点数。

## 2.2 感染二叉树需要的总时间

### 2.2.1 问题描述

给定一棵二叉树和一个起始感染结点 start，感染从第 0 分钟开始，之后每分钟会沿着相邻结点向外扩散，要求计算感染整棵树所需的分钟数。

### 2.2.2 基本要求

程序需要从输入中建立二叉树，并在树中模拟感染扩散过程，输出感染全部结点所需要的时间。

### 2.2.3 数据结构设计

Tree_node 结点中保存 parent、l_child、r_child 和 staute 四个信息。其中 parent 指针用于向上扩散，l_child 和 r_child 用于向下扩散，staute 用作访问标记，避免重复感染。

Tree 类用 vector<Tree_node *> 统一保存所有结点，便于根据编号直接定位结点并完成内存管理。整体建树方式相当于“数组下标 + 指针链接”的混合表示。

### 2.2.4 功能说明（函数、类）

- Tree_node：树结点，保存父结点、左右孩子和感染状态。
- Tree(int n)：构造函数，预留结点数组并初始化根结点。
- ~Tree()：析构函数，释放全部动态结点。
- main()：读入结点关系，建立父子联系，使用队列模拟感染扩散并输出结果。

### 2.2.5 调试分析（遇到的问题和解决方法）

这题的核心是把“树上的传播”转化为“图上的 BFS”。调试时最容易出错的地方是：

1. 需要同时考虑左孩子、右孩子和父结点三个方向；
2. 每轮感染后，只有当确实感染了新结点，时间才需要加 1；
3. 结点必须用状态标记，防止来回感染形成死循环。

程序通过队列逐层扩散，并在每一层统计是否有新结点被感染，从而准确得到总时间。

### 2.2.6 总结和体会

本题把树问题和 BFS 结合起来，体现了“层次扩散”模型的典型用法。程序中每个结点最多入队一次，时间复杂度为 O(n)，空间复杂度为 O(n)。

## 2.3 哈夫曼编码和译码

### 2.3.1 问题描述

给定一段 ASCII 文本，要求根据字符出现频率构造哈夫曼树，实现文件压缩、编码表显示和压缩文件译码。

### 2.3.2 基本要求

程序需要完成以下功能：读取文本、统计字符频率、构造哈夫曼树、生成编码、压缩文本、显示编码表、读取压缩文件并解压，同时对文件读写失败给出提示。

### 2.3.3 数据结构设计

Tree_node 结点保存左右孩子指针、字符内容和权值频率。Tree 类中使用 vector<Tree_node *> 存放当前所有结点，并通过 percent_list[256] 统计 ASCII 字符频率。

编码阶段通过遍历哈夫曼树生成字符到编码的映射表；译码阶段则利用同一棵树，从根结点开始按 0/1 逐步向左右孩子移动，直到到达叶结点后恢复原字符。

当前程序将压缩结果保存为 0/1 字符串，便于展示编码过程和验证译码正确性。

### 2.3.4 功能说明（函数、类）

- Tree_node：哈夫曼树结点，保存左右孩子、字符和频率。
- Tree：保存哈夫曼树结点集合，并提供 get_min() 等辅助功能。
- Tree::get_min()：在当前结点集合中查找权值最小的结点。
- Tree::deleteTree(Tree_node *node)：递归释放哈夫曼树。
- readTxtFile(const string &filename)：读取文本文件内容。
- writeStringToFile(const string &content, const string &filename)：把字符串写入文本文件。
- writeCompressedFile(const string &compressed, const string &filename)：保存压缩后的 0/1 字符串。
- readCompressedFile(const string &filename)：读取压缩文件内容。
- generateCodes(Tree_node *node, string code, vector<string> &codes)：递归生成哈夫曼编码表。
- compressFile(const string &content, Tree_node *root, const vector<string> &codes)：根据编码表生成压缩结果。
- decompressFile(const string &compressed, Tree_node *root)：根据哈夫曼树恢复原文。
- displayHuffmanCodes(const vector<string> &codes)：输出当前编码表。
- showMenu()：显示交互式菜单。
- main()：负责文件读取、频率统计、建树、编码、解码和交互式操作。

### 2.3.5 调试分析（遇到的问题和解决方法）

该题调试的重点有两个：一是编码和译码必须使用同一棵哈夫曼树，二是文件 I/O 过程要能正确处理打开失败等异常情况。为便于验证，程序在压缩后会输出编码预览，并在解压后将结果与原文进行比对。

另外，哈夫曼树构造时采用了不断查找最小权值结点的方式，逻辑清晰，便于检查每一步合并是否正确。为了提高可读性，程序还设计了菜单界面，使用户可以按需选择压缩、解压或显示编码表。

### 2.3.6 总结和体会

本题让我进一步理解了哈夫曼树“贪心合并最小权值结点”的思想，也熟悉了文件读取、字符串编码和树形递归遍历的综合应用。

复杂度方面，统计频率和编码、译码过程都与文本长度成正比，分别为 O(n) 或 O(m)，其中 n 为原文长度，m 为压缩后的比特串长度。当前实现中，构造哈夫曼树采用反复线性查找最小结点的方式，因此建树部分的时间复杂度为 O(k^2)，k 为不同字符数；若改用优先队列，复杂度可进一步降为 O(k log k)。

## 2.4 判断两棵树是否同构并计算深度

### 2.4.1 问题描述

给定两棵二叉树，要求判断它们是否同构。如果一棵树可以通过若干次左右孩子互换变成另一棵树，则称两棵树同构；同时需要输出两棵树各自的深度。

### 2.4.2 基本要求

程序需要从输入中建立两棵树，判断是否同构，并分别输出两棵树的深度。

### 2.4.3 数据结构设计

Tree_node 结点只保存左右孩子指针和字符标记 icon。Tree 类中用 vector<Tree_node *> 存放结点，并通过 root 记录根结点下标。

建树时先根据输入建立所有结点，再通过“是否被当作孩子”的标记数组确定根结点。深度计算采用递归方式，返回当前结点左右子树深度的最大值再加 1。

### 2.4.4 功能说明（函数、类）

- Tree_node：树结点，保存字符和左右孩子指针。
- Tree(int n)：构造函数，按结点数初始化结点数组。
- ~Tree()：析构函数，释放所有动态结点。
- Tree::calcDepth(Tree_node *node)：递归计算以 node 为根的树深度。
- Tree::get_tree_depth()：根据 root 计算整棵树深度。
- isIsomorphic(Tree_node *t1, Tree_node *t2)：递归判断两棵树是否同构。
- char2index(const string &s)：把输入中的结点编号字符串转换为整数，遇到 “-” 时返回 -1。
- main()：读入两棵树，完成建树、同构判断和深度输出。

### 2.4.5 调试分析（遇到的问题和解决方法）

这题的关键是正确处理“左右孩子互换后仍然算同构”的条件。调试时要特别注意两点：

1. 输入中孩子为空时用 “-” 表示，因此需要先转换为 -1 再判断；
2. 题目允许结点字符重复，不能只靠字符值判断结点是否相同，必须结合树结构递归比较。

程序先用标记数组找出根结点，再调用递归函数判断同构关系。这样可以避免因为根结点未知而导致遍历入口错误。

### 2.4.6 总结和体会

本题的重点在于树的结构比较，而不是单纯的遍历。通过递归同时考虑“同侧匹配”和“交叉匹配”，可以自然地完成同构判断。深度计算也可以借助同样的递归思想实现。整体时间复杂度与树规模同阶，空间复杂度主要来自递归调用栈。
