1. 涉及数据结构和相关背景

本实验使用哈夫曼树（最优二叉树）实现无损压缩。基本思想是：出现频率高的字符分配较短编码，出现频率低的字符分配较长编码，从而减少整体编码长度。

相关背景：
- 前缀编码：任何字符编码都不是另一个字符编码的前缀，因此解码不会产生歧义。
- 贪心算法：每次合并权值最小的两个结点，最终得到全局最优的编码树。
- 二叉树遍历：约定左分支为 0，右分支为 1，用递归方式生成编码表。

2. 实验内容

2.1 哈夫曼编码与译码

2.1.1 问题描述

读取文本文件，统计字符出现次数，构建哈夫曼树生成编码表，将原文转换为 01 串并保存；再根据哈夫曼树将压缩串还原为原文，并验证压缩和解压结果是否一致。

2.1.2 基本要求

- 能读取文本文件并统计字符频率。
- 能根据频率构建哈夫曼树。
- 能生成并显示哈夫曼编码表。
- 能将文本压缩为二进制字符串并保存。
- 能读取压缩文件并完成解码恢复。
- 能处理空文件、非法输入等基本异常情况。

2.1.3 数据结构设计

哈夫曼树节点保存字符信息、权值以及左右孩子指针：

```
Tree_node
┌────────────────────┐
│ l_child  左孩子指针 │
│ r_child  右孩子指针 │
│ content  结点字符   │
│ percent  出现频率   │
└────────────────────┘
```

程序先统计每个 ASCII 字符的频率，再把出现过的字符作为初始叶子结点放入结点列表。随后不断取出权值最小的两个结点合并，直到只剩一个根结点。编码时约定左边为 0，右边为 1。

2.1.5 调试分析（遇到的问题和解决方法）

| 问题 | 现象 | 原因 | 解决 |
|------|------|------|------|
| get_min() 重复调用 | 合并结点时树结构异常 | erase 之后索引发生变化 | 先保存最小结点指针，再执行删除 |
| 父结点未加入列表 | 哈夫曼树无法继续合并 | 合并后遗漏 push_back | 创建父结点后立即加入结点列表 |
| 单字符文件处理异常 | 编码为空或解码循环 | 只有一个叶子结点时路径长度特殊 | 对单字符文件单独处理，直接赋值编码 |
| 压缩和解压结果不一致 | 结果文本和原文不同 | 编码表或树结构错误 | 重新检查建树、编码生成和解码路径 |
| 内存释放问题 | 程序结束时崩溃 | 重复释放树结点 | 只通过根结点递归释放一次 |
